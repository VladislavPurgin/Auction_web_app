# Auction_web_app Опис проєкту
Тестове завдання: Реалізація сервіс на FastAPI з підтримкою WebSocket, який дозволяє:
* Створювати лоти для аукціону.
* Робити ставки через REST.
* Отримувати оновлення ставок у реальному часі через WebSocket.

Проєкт має такий функціонал:
* REST API: Є можливість створювати лоти, здійсненювати ставки, переглядати активні лоти.
* WEBSockets: Отримуємо оновлення про нові ставки в реальному часі.
* Контейнеризація: Повністю налаштоване середовище Docker (App + PostgreSQL).

● Ендпоінти REST:
* POST /lots — створити лот.
* POST /lots/{lot_id}/bids — зробити ставку.
* GET /lots — список активних лотів.
● WebSocket:
* GET /ws/lots/{lot_id} — підписка на події лота.

## Вимоги
Перед запуском переконайтеся, що у вас встановлені:
Docker (версія 20.10+)
Docker Compose (версія 2.0+)
Git

Також необхідні бібліотеки для роботи сервісу знаходяться у файлі requirements.txt. При запуску через Docker вони встановлюються автоматично.

## Швидкий старт проєкту
1. Клонуємо репозиторій
   ```git clone https://github.com/vlad-purhin/Auction_web_app.git```
   
   ```cd Auction_web_app```

3. Запуск сервіса через docker
   docker-compose up --build

Сервіс буде доступний за адресою: http://127.0.0.1:8000

3. Тестування
Для перевірки роботи сервісу можна використовувати Postman або інтерактивну документацію Swagger: http://127.0.0.1:8000/docs.

Створення лота
Відправте POST запит на /lots:

JSON
{
  "title": "Iphone 17",
  "description": "New model Iphone",
  "start_price": 1500.0
}
Перегляд списку лотів
Відправте GET запит на /lots, щоб отримати список активних лотів.

Здійснення ставки
Відправте POST запит на /lots/{lot_id}/bids (наприклад, для id: 1):

JSON
{
  "bidder_name": "Vlad",
  "amount": 1600
}
WebSocket оновлення
Підписавшись через WebSocket на конкретний ID лота, ви будете отримувати миттєві сповіщення про нові ставки у форматі:

JSON
{
  "type": "bid_placed",
  "lot_id": 1,
  "bidder": "Maria",
  "amount": 1700.0
}

Змінні середовища
Для локальної роботи використовується файл .env. База даних автоматично ініціалізується з параметрами:

DB_USER: user

DB_PASSWORD: 123456

DB_NAME: auction_db
